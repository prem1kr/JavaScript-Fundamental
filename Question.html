Q1. Write a JavaScript function called outerFunction that takes a parameter and returns an inner function. The inner function should access both the parameter of outerFunction and a variable declared within outerFunction. Demonstrate how lexical scoping allows the inner function to maintain access to these variables even after outerFunction has finished executing.

code:-
function outerFunction(outerParameter) {
const outerVariable = "Hello from outerfunction";
function innerFunction() {
console.log(outerVariable);
console.log(outerParameter);
}
return innerFunction; }
const myInnerFunction = outerFunction("Passed Parameter");
myInnerFunction();

Explanation:-
1. outerFunction(outerParam):-
  • This function takes one parameter outerParam.
  • Inside outerFunction, a variable outerVariable is declared.
    • 2. innerFunction():-
  • This is a nested function (also known as the inner function) declared inside outerFunction.
  • innerFunction has access to both the outerParam passed to outerFunction and the outerVariable declared within outerFunction.
    • 3. Returning innerFunction:-
  • outerFunction returns innerFunction, allowing you to store it in a variable (myInnerFunction).
    • 4. Lexical Scoping:-
  • When outerFunction is called and returns innerFunction, the innerFunction still has access to the outerVariable and outerParam even after outerFunction has finished executing. This is because of lexical scoping in JavaScript, where functions remember the scope in which they were created.

    } else {
    return null; }
    } const datePattern = '(\\d{2})-(\\d{2})-(\\d{4})';

    • Q5. You are building a shipping application. Write a program that takes the type of package ("standard", "express", or "overnight") and uses a switch statement to calculate and print the estimated delivery time based on the package type. For example, "standard" might take 3-5 days, "express" 1-2 days, and "overnight" would be delivered the next day.

    code:-
    function getDeliveryTime(packageType) {
    let deliveryTime;
    switch (packageType.toLowerCase()) {
    case "standard":
    deliveryTime = "3-5 days";
    break;
    case "express":
    deliveryTime = "1-2 days";
    break;
    case "overnight":
    deliveryTime = "Next day";
    break;
    default:
    deliveryTime = "Unknown package type";
    break; }
    console.log(`Estimated delivery time for ${packageType} package: ${deliveryTime}`); }
    getDeliveryTime("standard");
    getDeliveryTime("express");
    getDeliveryTime("overnight");
    getDeliveryTime("same-day");

    Explanation:-
    1. getDeliveryTime(packageType):-
  • This function takes the packageType as input and determines the delivery time based on the type of package.
  • The switch statement is used to evaluate the packageType and assign the appropriate deliveryTime based on the type:
  • "standard": Delivery time is 3-5 days.
  • "express": Delivery time is 1-2 days.
  • "overnight": Delivery time is Next day.
  • The toLowerCase() method is used to handle any case variations in the input.
  • If the packageType does not match any of the cases, the default case sets deliveryTime to "Unknown package type".
  • The function then prints the estimated delivery time.
    • 2.Testing the Function:
  • The function is tested with different package types: "standard", "express", "overnight", and an unknown type "same-day".
